preconstructor

protected void preconstructor()
Executed immediately after the call to JFrame's constructor via super(). This method should be used to set up any variables that need to be non-null before anything else executes. It is not necessary to implement this method. It is primarily available to resolve strange issues such as class-global arrays being null despite assignment at their declaration.
